Equality ==

Type coercion takes place if operands are not of the same type.

If both operands are objects, then the internal references are compared.

Useful: to check whether value is null or undefined, use null == val

All rules of comparison are not worth to remember. See 11.9.3 The Abstract Equality Comparison Algorithm. And see example.

Identity or Strict Equality ===

If types are different, return false

For numbers, if any operand is NaN, return false; +0 === -0

null !=== undefined

Objects are compared by reference

Full set of rules: 11.9.6 The Strict Equality Comparison Algorithm.
